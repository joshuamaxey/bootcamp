Box Model

Box Properties

Every HTML element is contained within a box. This box includes the content (center), the padding around the box, the border around the padding, and the margin around the border. Based on the box type (box-sizing property), the width and height properties of the box may refer to the content, padding, border, and margin of the box together OR only to the content of the box.

  • width
  • height
  • pading (top, right, bottom, left)
  • border (top, right, bottom, left)
  • margin (top, right, bottom, left)

Box Types

The 'box-sizing' property determines whether the 'width' and 'height' properties of the box refer to the content, padding, border, and margin all together (border-box) OR just to the content of the box (content-box)

  • box-sizing: content-box (default)
  • box-sizing: border-box

Display Types

The 'display' property determines whether an element is treated as a block or an inline box, and also determines the layout of its children (inline, block, flex, grid, etc)

  • display: block
  • display: inline (NOTE: an inline div will automatically adjust to the width of its contents by default)
  • display: inline-block
  • display: flex
  • display: grid

Positioning

Positions

The 'position' property allows us to manipulate the location of an element on our page. Some positions align with the 'flow' of the page, while others cause an element to be removed from the 'flow' and allow it to be positioned independently of other elements. The 'flow' of a page refers to the relationship between block and inline elements before any changes are made to the layout of the page or the position of individual elements.

  • position: static

      • 'static' is the default position. A static element will apear normally according to the page flow, so a static element is NOT considered a 'positioned' element. The top, right, bottom, left, and z-index 'properties' do not affect static elements

  • position: relative

      • Elements that have a 'relative' position will remain in their original position in the page flow by default (like a static element). However, relative elements can be offset from that position using the top, right, bottom, and left properties. The element is positioned RELATIVE to its ORIGINAL PLACE in the page flow according to these properties.

      Note that using relative positioning creates a STACKING CONTEXT, potentially causing elements to overlap. The position of overlapped elemements is determined using the z-index property. A higher z-index (for example: 1) moves an element closer to the user, while a lower z-index (for example: -1) moves an element away from the user. The default z-index for all elements is 0.

  • position: absolute

      • An element with an 'absolute' position is removed from the page flow such taht other elements behave as if it does not exist. By default, the element is positioned in relation to its CLOSEST POSITIONED ANCESTOR. If there is no ancestor, it is positioned relative to the HTML DOCUMENT ITSELF. Absolutely positioned elements can be offset from their default position using the top, right, bottom, and left properties

      Note that using absolute positioning also creates a STACKING CONTEXT

      Note that it is common practice to set the position of an element's CONTAINER ELEMENT to 'relative' so that the position of absolutely-positioned child elements is determined based on the position of the container.

  • position: fixed

      • A 'fixed' element is removed from the page flow, similarly to an absolutely positioned element. However, a fixed element's position is always determined relative to the HTML DOCUMENT ITSELF, and NOT to any ancestor element. A fixed element's position can be manipulated usign the top, right, bottom, and left properties.

      Note that fixed positioning also creates a STACKING CONTEXT.

      Note that a fixed element will remain in the same spot on the page REGARDLESS of the WINDOW SIZE or USER SCROLLING.

  • position: sticky

      • A 'sticky' element remains in its original position according to the page flow, and it is positioned relative to its closest block-level ancestor AND any scrolling ancestors. Essentially, it behaves like a relatively-positioned element until the point at which a user scrolls past it in the viewport. At that 'scrolling point', the element 'sticks' to the page at whatever location it has been positioned using the top, left, bottom, and right properties.

      Note that 'sticky' elements also create a STACKING CONTEXT.

      Note that at least ONE threshold value (top, right, bottom, or left) must be used in order for sticky positioning to work. A sticky element begins as a 'relative' element until at least one threshold value is applied and the user scrolls past its original position, at which point it 'sticks' to the position specified using the top, right, bottom, and left properties.

Position Properties

The top, right, bottom, left, and z-index 'properties' are used to manipulate the location of an element on the page based on that element's 'position' (static, relative, absolute, fixed, or sticky).

  • top
  • right
  • bottom
  • left
  • z-index

Flexible Box Model

Basic Flexbox Properties

By default, the flexbox container is automatically sized based on the size of the viewport. Elements within the container are resized and distributed to fill the available space.

Flex Container Properties

  • display:

      • flex

      inline-flex

  • flex-wrap:

      • wrap

      nowrap

      wrap-reverse

  • flex-direction:

      • row

      row-reverse

      column

      column-reverse

  • flex-flow:

      • row wrap (etc)

      This is a shorthand property that combines the flex-wrap and flex-direction properties

Flex Item Properties

  • justify-content:

      • Defines the alignment of flex items along the MAIN AXIS and distributes extra container space around and between items (Main axis is horizontal by default)

    • flex-start

      • Stack items at the start

    • flex-end

      • Stack items at the end

    • center

      • Stack items in the middle

    • space-between

      • distribute extra space between items

    • space-around

      • Distribute space before, between, and after items

    • stretch

      • Grow or shrink items to fill the container

  • align-items:

      • Defines the alignment of flex items along the CROSS AXIS and distributes extra container space around and between items (Cross axis is vertical by default)

    • flex-start
    • flex-end
    • center
    • stretch
    • gap

      • defines the amount of space between items

      (REMINDER: 'padding' controls the amount of space before the first item and after the last item)
