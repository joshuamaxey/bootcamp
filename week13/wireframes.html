Wireframe Specs

Wireframes are lighweight, rapidly developed design specs that ourline the structure and layout of a web page. Wireframes are a kind of 'spec.'

As developers, we have to be able to identify HTML elements from a wireframe spec and then use that understanding to generate semantic HTML code.

NOTE: Semantic means, "relating to meaning in language or logic".

HTML, CSS, JS

Webpages have THREE major components:

  • HTML: Page structure and layout
  • CSS: Page style and positioning
  • JavaScript: Page functionality, utility, and interactivity

Reading Wireframes

Wireframes are rough structural plans from which we can deduce which HTML elements will be present in our webpage. They provide structure without the style. HTML elements can be moved around easily with CSS after the structure has initially been laid out.

Using this wireframe, we can identify and label the semantic HTML tags for each element of the page.

Once we have identified all of the HTML elements, we can write out our HTML code. Notice that there is NO styling yet:
