What is Web Accessibility?

Web is the practice of implementing design decisions which make a website accessible to a variety of users.

Typically, we think of web accessibility as supporting users with disabilities

When we design with accessibility in mind, we create a better user eperience for all users.

Federal law requires that websits be designed to meet certain minimum criteria for accessibility.

WCAG Compliance

The WCAG (Web Content Accessibility Guidelines) is a set of standards for designing accessible web content.

Four Principles

The following WCAG Quick Reference Guide provides a full list of guidelines that will help you to meet the WCAG according to four primary principles while the WCAG Checklist offers a short summary of these guidelines.

Percievable

Information and user interface components must be presentable to users in ways they can percieve.

  • EXAMPLE 1: Structure HTML such that the code reflects a hierarchy of relationships between elements which can be interpreted by assistive devices.
  • EXAMPLE 2: When Color is used to convey meaning, convey that maning through other means as well so that colorblind users can percieve the meaning as well as those who can percieve color.
  • EXAMPLE 3: Avoid using images of text, as these cannot be interpreted by assisstive devices such as screenreaders. Use text instead.

Operable

User interface components and navigation must be operable.

  • EXAMPLE 1: Users should be able to perform all operation using a keyboard, as opposed to relying solely on a mouse. This supports users with mobility limitations.
  • EXAMPLE 2: All navigation links should be clearly labeled as to their purpose(usually through the text that is displayed).
  • EXAMPLE 3: Targets (buttons, hyperlinks, etc) must be sufficiently large that they are user-friendly.

Understandable

Information and the operation of the user interface must be understandable.

  • EXAMPLE 1: The web page must specify its language and support translation for multilingual users.
  • EXAMPLE 2: Clear error messages must be displayed to assist the user when there are errors in user input.
  • EXAMPLE 3: Predictable navigation patterns must exist across web pages as well as clearn instructions or directions for user intraction.

Robust

Content must be sufficiently robust such that it can be interpreted by a wide variety of user agents, including asestive technologies.

  • EXAMPLE 1: All elements have unique id attributes. This supports parsing and interpretation of the contents by assistive devices.
  • EXAMPLE 2: All HTML syntax is correct, with appropriate nesting and opening/closing tags
